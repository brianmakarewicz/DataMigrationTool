Canonical object catalog agreed 2026-07-02

One object = one upload unit — one FBDI ZIP, one HDL .dat submission, or one FBL ZIP, submitted to one Fusion import job. Nothing more, nothing less. Sub-tables inside a ZIP (Customer's 7 CSVs, Grants' 15 CSVs, Worker.dat's 7 METADATA sections) are sub-objects, never separate objects. Every registry, package, folder, doc, and UI label must use these codes and display names. 46 objects total: 24 FBDI + 14 HDL + 7 FBL + 1 web service. The compact list lives in Design Reference §14; this page is the full lineage detail.

Naming convention. CEMLI code = PascalCase, no spaces (SupplierSiteAssignments). Display name = the code with spaces (Supplier Site Assignments). Packages: DMT_{ABBREV}_{FBDI|HDL|FBL}_GEN_PKG, DMT_{ABBREV}_VALIDATOR_PKG, DMT_{ABBREV}_RESULTS_PKG. One objects/{CemliCode}/ folder per object — sub-object folders get folded into their parent. Decided 2026-07-02: GLBudgets (not GLBudgetBalances), MiscReceipts, Grants.
How to read the lineage tables. Upload CSV = the source file the upload UI stages (from DMT_UPLOAD_OBJECT_TBL, which keys on its own table-level codes — it gets a CEMLI_CODE column in the rename sweep). FBDI file(s) = the interface files the generator writes inside the ZIP. Table names omit the DMT_ prefix and _TBL suffix. The four identifier columns show how each registry spells the CEMLI: ✓ = identical to the canonical code · — = object not present there · a bold value = spelled differently (a rename-sweep target). A bold ZIP prefix likewise deviates from the {CemliCode}_ convention.

Shared-stage packages are intentional where objects share staging tables (Blankets/Contracts on PO tables, 1099 on AP tables, supplier children transformed/reconciled with the header) — but per the error-attribution rule, shared stages must still write errors against the correct object's rows. Upload naming convention — dynamic header upload

RuleConvention (enforced by DMT_CSV_UPLOAD_PKG / DMT_UPLOAD_DICT_TBL)
Single-CSV uploadUser picks the object on the page — the file can be named anything; p_object_code drives the target table.
ZIP bundle uploadEach file inside the ZIP must be named exactly {OBJECT_CODE}.csv (case-insensitive match on DMT_UPLOAD_OBJECT_TBL.CSV_FILENAME, e.g. POZ_SUPPLIERS.csv, PO_HEADERS.csv). Unmatched files are skipped and reported.
Header row (dynamic)Row 1 must contain STG-table column names — uppercased and matched against the uploadable columns in DMT_UPLOAD_DICT_TBL (seeded from USER_TAB_COLUMNS). Any column order, any subset; unknown headers are ignored with a warning; zero matches = hard error.
Dictionary upkeepAfter any STG DDL change run DMT_UPLOAD_DICT_PKG.REFRESH_DICTIONARY — otherwise new columns aren't uploadable.
GapUpload object codes are table-level (POZ_SUPPLIERS), a different code system from CEMLI codes. Rename sweep adds DMT_UPLOAD_OBJECT_TBL.CEMLI_CODE so every upload row maps to one canonical object — and adds the missing Requisitions + ProjectBudgets rows so their CSVs are uploadable at all.
